# 手动启动服务指南

如果自动启动脚本出现问题，可以手动启动服务。

## 方法一：使用两个终端窗口

### 终端1 - 启动后端服务

```powershell
cd backend
npm start
```

**成功标志**：看到类似以下信息
```
服务器运行在 http://localhost:3000
数据库已连接
```

### 终端2 - 启动前端服务

```powershell
cd frontend
npm run dev
```

**成功标志**：看到类似以下信息
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

然后在浏览器访问：http://localhost:5173

## 方法二：使用启动脚本（推荐）

### Windows

双击项目根目录的 `start.bat` 文件

或运行：
```powershell
.\start.ps1
```

### 检查服务状态

运行检查脚本：
```powershell
.\scripts\windows\check_services.ps1
```

## 常见问题

### 1. 端口被占用

如果看到端口被占用的错误，可以：

**Windows:**
```powershell
# 查看占用端口的进程
netstat -ano | findstr :3000
netstat -ano | findstr :5173

# 结束进程（替换PID为实际进程ID）
taskkill /F /PID <PID>
```

### 2. 依赖未安装

如果看到 `Cannot find module` 错误：

```powershell
# 安装后端依赖
cd backend
npm install

# 安装前端依赖
cd ../frontend
npm install
```

### 3. 数据库初始化失败

删除 `backend/database/mms.db`（如果存在），然后重新启动后端服务，系统会自动初始化数据库。
